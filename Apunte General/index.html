Clase #1

¿Qué son y como funcionan los sitios web?

Un sitio web es un conjunto de páginas web relacionadas que están diseñadas y organizadas para ser accedidas a través de Internet. Es una forma común de presentar información, contenido y servicios en línea.
La estructura básica de un sitio web se compone de archivos de hipertexto escritos en lenguajes como HTML, CSS y JavaScript. Estos archivos contienen el código que define la estructura, el diseño y la funcionalidad del sitio web. Cuando un usuario ingresa la dirección de un sitio web en su navegador, se establece una solicitud al servidor web correspondiente. El servidor procesa la solicitud y envía los archivos necesarios para mostrar la página web solicitada en el navegador del usuario. El navegador interpreta el código HTML, CSS y JavaScript y muestra el contenido y el diseño de la página web. Los sitios web pueden variar en su complejidad y funcionalidad. Pueden ser estáticos, con contenido fijo que no cambia con frecuencia, o dinámicos, con contenido generado en tiempo real, interacciones de usuarios, bases de datos, y muchas formas más. Y las temáticas, ¡son infinitas! Para profundizar en este tema, los y las invitamos a ver la siguiente presentación.



Una página web es una página virtual con información dentro de ella. Cada una tiene diferente contenido en su interior, y cumple una función diferente o aporta diferente información según ese contenido. Ae puede acceder a una página web desde un navegador web como Chrome, Opera, Firefox, etc.

Para ver la estructura digital de un sitio web podemos hacer click derecho con el mouse e ir a la opción de Inspeccionar. Ahí podemos ver que la estructura de una página web es texto, lenguaje de programación que el navegador procesa y lo interpreta en lo que nosotros vemos en la página web.
Además de contener diferentes tipos de texto, un sitio web puede contener elementos interactivos, como imágenes, enlaces, estilos, videos, formularios, botones, etc. Sin mencionar que también pueden contener funciones interactivas y dinámicas utilizando lenguajes de programación como JavaScript.

Para crear una maqueta de una página web se usan dos lenguajes de programación, HTML y CSS. HTML significa HyperText Markup Language, nos permite estructurar nuestro sitio, darle forma, insertar los elementos que deseamos. CSS significa Cascading Styles Sheet, nos permite darle estilo al sitio web, como tamaño de texto, márgenes, bordes, etc. También hay un tercer lenguaje, JavaScript, un lenguaje que tiene un gran uso, le otorga dinamismo y complejismo.



Al ver el código fuente de una página, su estructura, vemos muchas líneas que tardaríamos muho en escribir a mano, por eso existen los IDE, Integrated Development Environment, programas que nos ayudan a escribir código en distintos lenguajes.
Los textos de código que escribimos se guardan en archivos, todos con una extensión específica según el tipo de lenguaje que usemos. Por ejemplo: para un PDF añadimos la extensión .pdf, para un archivo html usamos .html.

Al crear una página web, seguis una serie de pasos. Pensas en lo que querés que haga, la diseñas, haces un modelo de prueba, y ahí es cuando la pasas a una página real.

HTML semántico

El HTML semántico es una práctica de estructuración y etiquetado de código HTML que busca dar un significado y contexto semántico a los elementos y contenido de una página web. Se utilizan etiquetas específicas que describen el propósito y la función de cada elemento.
El HTML semántico tiene varios beneficios:

  • Mejora la accesibilidad: Al utilizar etiquetas semánticas apropiadas, se facilita la comprensión de la estructura y el contenido de una página web para los usuarios que dependen de tecnologías de asistencia, como lectores de pantalla.

  • Mejora el SEO: Los motores de búsqueda utilizan la estructura semántica del HTML para comprender y clasificar el contenido de una página web. Al utilizar etiquetas semánticas, se facilita la indexación y el posicionamiento en los resultados de búsqueda.

  • Facilita el mantenimiento y la colaboración: Un código HTML semántico y bien estructurado es más fácil de mantener y entender, lo que facilita el trabajo en equipo y la colaboración en proyectos web.

  • Mejora la legibilidad y comprensión del código: Al utilizar etiquetas semánticas, el código HTML se vuelve más descriptivo y comprensible, lo que facilita su lectura y comprensión tanto para desarrolladores como para otros interesados en el proyecto.


Diseño Web

El diseño web y el desarrollo web son dos aspectos diferentes pero complementarios en la creación de sitios web. El diseño web se centra en la apariencia visual y la experiencia de usuario de un sitio web. Los y las diseñadoras se encargan de crear la estructura, el diseño gráfico, la disposición de los elementos, los colores, las tipografías y otros aspectos visuales del sitio. Utilizan herramientas de diseño gráfico y prototipado para crear el aspecto visual de las páginas web. Mientras que el desarrollo web se enfoca en la implementación técnica y funcionalidad de un sitio web. Los y las desarrolladoras web trabajan con lenguajes de programación como HTML, CSS y JavaScript para construir y programar el sitio web. Se encargan de la estructura del sitio, la interacción con el usuario, la funcionalidad de los formularios, la conexión con bases de datos y otros aspectos técnicos. El objetivo principal del desarrollo web es hacer que el sitio web funcione correctamente y cumpla con los requisitos y funcionalidades establecidos.

Un paso muy importante es construir un wireframe, un diseño que nos permite tener una vista general de como se verá el sitio web, y de esa manera realizar correciones menores de manera rápida y precisa. Un sitio muy bueno para hacer esto es Figma, en él hacer un wireframe es mucho más fácil que a mano.

Clase #2

Bloques vs líneas

En HTML existen dos tipos principales de etiquetas: las etiquetas de línea (inline) y las etiquetas de bloque (block). Estas etiquetas determinan cómo se comporta y se muestra el contenido dentro de un documento HTML pero no tienen una función específica, sino que sólo sirven como contenedores de más elementos. La diferencia existe en el espacio que ocupan en la pantalla, y en las características que se les agregue en CSS, cómo margin o padding.
Es importante tener en cuenta que la visualización de las etiquetas de línea y de bloque puede ser modificada mediante estilos CSS, lo que permite personalizar su apariencia y comportamiento.

Etiquetas de línea: Las etiquetas de línea son utilizadas para envolver y aplicar estilos a contenido dentro de un flujo de texto. No generan saltos de línea y se muestran en línea con el texto circundante. Estas etiquetas son ideales para aplicar estilos a partes específicas del texto, como resaltar una palabra, agregar enlaces o incluir imágenes dentro de una oración.
Etiquetas de bloque: Las etiquetas de bloque generan un bloque de contenido que ocupa todo el ancho disponible en su contenedor. Por defecto, cada etiqueta de bloque comienza en una nueva línea y crea un espacio vertical entre elementos adyacentes. Estas etiquetas se utilizan para estructurar y agrupar secciones de contenido en un documento HTML. Pueden contener tanto etiquetas de línea como otras etiquetas de bloque.

Para cambiar un elemento en línea a uno en bloque y viceversa se usan las propiedades de CSS: display: block y display: inline.

¿Cómo se escribe CSS?

CSS (Cascading Style Sheets) es un lenguaje de estilo utilizado para describir la presentación y el diseño de un documento HTML o XML. Se utiliza para definir cómo se ven los elementos y componentes de una página web, incluyendo el color, la tipografía, el tamaño, el espaciado, el diseño y otros aspectos visuales.
Por ejemplo:

  • Para subrayar usamos text-decoration: underline;

  • Para añadir un borde usamos border: 2px solid red;

Varias rpopiedades requieren colores, y algumos sitios para configurar estos colores son:

  • colorhunt

  • colorhail

  • color Adobe

  • canva colors

  • blend

  • htmlcolorcode

Algunas propiedades de texto son:

  • font-weight: define el grosor de las letras del texto.

  • font-style: indica el estilo del texto, como normal, cursiva o inclinada.

  • text-align: controla la alineación del texto en su contenedor, ya sea izquierda, derecha, centro o justificado.

  • text-decoration: permite agregar decoraciones al texto, ya sea subrayado, tachado o resaltado.

  • text-transform: cambia la capitalización del texto, como mayúsculas, minúsculas o capitalización inicial.

  • line-height: establece la altura de la línea de texto.

  • letter-spacing: controla el espaciado entre carácteres del texto.

Al insertar una imagen de fondo con background-image, hay una propiedad a tener en cuenta, background-repeat, especifica si la imagen de fondo se repitey como lo hace. Sus valores son:

  • repeat, la imagen se repite vertical y horizontalmenet.

  • repeat-x, la imagen se repite solo horizontalmente.

  • repeat-y, la imagen se repite solo verticalmente.

  • no-repeat, la imagen no se repite.

También está background-position, especifica la posición inical de la imagen de fondo. Sus valores son:

  • Píxeles o porcentajes

  • left

  • right

  • top

  • bottom

  • center

También está background-size, especifica el tamaño de la imagen de fondo. Sus valores son:

  • Píxeles

  • Porcentajes

  • auto

  • cover, la imagen cubre completamente el área.

  • contain, la imagen se ajusta correctamente dentro del área sin recortarse.

También está background-attachment, especifica si la imagen de fondo se desplaza con el contenido de la pantalla o si se mantiene fija en su posición. Sus valores son:

  • scroll, la imagen se desplaza con el contenido.

  • fixed, la imagen se mantiene fija en su posición al desplazar el contenido de la página.

  • local, la imagen se desplaza junto con el contenido de su contenedor, sin salirse de él nunca.

Posicionamientos

La propiedad z-index se usa para contrlar la posición de los elementos en el eje z. Su valor es un número entero, los que tienen el valor más alto se superponen a los demás, al tener el mismo valor se ordenan en el orden en el que hayan sido insertados en el código.


La propiedad de CSS position se usa para especificar la posición de un elemento en relación con los demás elementos. En este caso se usan también las propiedades top, bottom, left, right. Sus valores son:

  • static, valor por defecto, aceptan el flujo normal del documento.

  • relative, los elementos se posicionan en relación con su posición normal.

  • absolute, los elementos se posicionan en relación con el primer elemento padre que tenga un valor distinto a static. y si no hay, usa el body como referencia.

  • fixed, los elementos se posicionan en relación con la ventana del navegador y se mantienen en la misma posición incluso al desplazar la página.

  • sticky, los elementos se comportan como relative hasta que alcanzan una posición específica en el navegador, ahí se vuelven fixed.

Para centrar un elemento con top, bottom, left, right, hacemos lo siguiente. Tomamos la mitad del ancho del elemento, (100 % 2 = 50), y usamos la propiedad calc() en otra de las propiedades anteriores.
top: 0:
left: calc(50% - 50px);

Grillas: construcción, partes y espaciados.

Para armar una grilla primero insertamos display: grid; en el contenedor.
Luego para asignar las columnas usamos grid-template-columns: 100px 100px 100px;
Luego para asignar las filas usamos grid-template-rows: 50px 150px 100px 50px 20px;
Luego para definir el espacio entre columnas y filas usamos grid-column-gap: 20px; y grid-row-gap: 10px;.

Para asignar elementos dentro de las grillas usamos las siguientes propiedades teniendo en cuenta las líneas de la grilla.
p{
grid-column: 3/4;
grid-row: 2/3;
}

Medidas de las grillas.

En vez de pixeles, también se pueden usar porcentajes para las medidas de las grillas.
grid-template-columns: 25% 50% 25%;

También se usa fr como una unidad de medida en grids. Es una forma de decir "free", se maneja en fracciones.
p{
width: 700px;
grid-template-columns: 300px 300px 1fr;
}

También tenemos grid-area: 1 / 1 / 2 / 7;.
El primer 1 significa primera línea horizontal.
El segundo 1 significa primera línea vertical.
El 2 significa segunda línea horizontal.
El 7 significa séptima línea vertical.

Clase #8

Pseudo-clases

Las pseudo clases son palabras clave que se usan en CSS para especificar estilos que aplican en ciertos momentos o acciones. Por ejemplo para especificar qué pasa cuando el cursor está encime del elemento, usamos:
button:hover {
background-color: blue;
}

Transform

Transform es una propiedad de CSS que se usa junto a hover. Su función, como dice que el nombre, es transformar el elemento, ya sea con el tamaño, la posición, o la rotación.

  • transform: scale(2); multiplica el tamaño del elemento por el número que se inserte.

  • transform: translate(20px, 30px); traslada el elemento según los valores que pongas. También está translateX() y translateY().

  • transform: rotate(45deg); rota el elemento según los grados(degrees) que insertes en la propiedad. Para cambiar el punto de rotación del centro, usamos transform-origin con dos valores, top o bottom, y left o right.

Transition

La transición es el tiempo que tardan en ejecutarse los cambios en el elemento por la propiedad transform.
La propiedad lleva tres valores: la propiedad que se va a ser afectada, los milisegundos que dura la transición, y la aceleración.
.caja:hover {
transform: rotate(30deg);
}
.caja {
transition: transform 200ms ease;
}
